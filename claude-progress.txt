## How to Use

### Mock mode (default, no backend needed):
```
VITE_BACKEND_MODE=mock pnpm dev
```

### Live mode (requires PostgreSQL + backend):
```
# Terminal 1: Start PostgreSQL
docker compose up -d db

# Terminal 2: Start backend
cd backend && source .venv/bin/activate && uvicorn app.main:app --reload --port 8000

# Terminal 3: Start frontend
VITE_BACKEND_MODE=live pnpm dev
```

## Architecture Notes
- ClaudeCodeRunner uses asyncio.create_subprocess_exec to spawn Claude CLI (not claude_agent_sdk)
- stream-json mode for tasks (real-time progress), json mode for chat (single-turn)
- --bare flag on task subprocesses (skip hooks/LSP/MCP for faster headless execution)
- Queue + background reader task pattern for clean async generator teardown
- PriorityScheduler is stateless (pure function: list in → list out)
- Orchestrator.start() resets stale RUNNING tasks to QUEUED on restart
- Three queue types: auto→done, semi→review (wait for approval), human→done (manual dispatch only)
- Frontend in 'live' mode is purely driven by WebSocket (backend is source of truth)
- Frontend in 'mock' mode runs original MockSimulator (no backend needed)
- Task runners execute in parallel asyncio tasks, each consuming a CLI subprocess stream
